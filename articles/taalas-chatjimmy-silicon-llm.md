## Taalasとは

Taalasは「1モデル = 1専用チップ」を作るスタートアップ。

どういうことかというと、例えばLlama 3.1 8Bを動かすための**専用チップ**を作る。そのチップではLlama 3.1 8B以外は動かない。多少のLoRAファインチューニングは可能だが、別のモデルは載らない。

**スペック（公式ブログより）：**
- 17,000 tokens/sec per user
- 現行SOTAの約10倍の速度
- 構築コスト1/20
- 消費電力1/10
- 3bit/6bitの積極的な量子化

**チームと資金：**
- チーム24人
- 2億ドル以上の資金調達
- うち使ったのは3,000万ドル

参考: [The Path to Ubiquitous AI - Taalas公式ブログ](https://taalas.com/the-path-to-ubiquitous-ai/)

## ポジショニング：さらにその先へ

前回のスペクトラムを拡張すると、こうなる。

```
柔軟性 高い ←――――――――――――――――――――――――――→ 低い
           GPU    TPU    Cerebras/Groq    Taalas

【GPU (NVIDIA)】
  完全汎用。何でも動く。CUDAエコシステム。

【TPU (Google)】
  テンソル計算に特化した半汎用チップ。モデルは自由に入れ替え可能。

【Cerebras / Groq】
  AI推論に特化したアーキテクチャ。でもモデルは入れ替え可能。

【Taalas】
  特定のモデルをシリコンに焼く。究極の特化。
```

Taalasはスペクトラムの**最も右端**にいる。ASICよりもさらに特化している。

メモリと計算をDRAM密度レベルで同一チップに統合し、HBMも先端パッケージングも液冷も3D積層もいらないと主張している。これはProcessing-in-Memory（PIM）と呼ばれるアプローチで、通常はプロセッサとメモリの間でデータを往復させるところを、メモリの中で直接計算してしまう技術だ。研究自体は以前からあるが、量産レベルの実例は限られていて、かなり大胆な技術的主張だ。

**トレードオフは明確：** モデルを焼き込むから速い。でもモデルを変えられない。

## ChatJimmyを触ってみた

Taalasが公開しているデモ「ChatJimmy」を実際に触ってみた。

https://chatjimmy.ai/

テスト用の質問はClaudeに考えさせて、出力もClaudeに評価してもらった。

### 日本語テスト：完全崩壊

「日本の少子高齢化問題について2000字程度で解説してください」と聞いてみた。

**速度：0.029秒で生成 / 15,821 tok/s**

速い。速いのは本物。で、出てきた日本語がこれ。

> 日本では、少子高齢化問題が大きな問題となっています。少子高齢化問題とは、子ども層が高齢化することなくて、高齢化問題を指摘することで生じていることです。です。原因:低い層の貧困さが深く、問題を残してます。低い層の資源が乖しく、高い資源に乖くことです。低い層の資源が高く、乖することです。低い層の高齢化が高く、高齢化が問題を高く問題することで生じています。

全文はもっと長いが、ずっとこの調子。「乖する」「問題を高く問題する」「生えしています」——日本語として成立していない。同じフレーズが無限ループのように繰り返される。

**日本語は使い物にならない。**

### 英語テスト：まともに動くが...

同じテーマを英語で聞いてみた。"Write a detailed essay about Japan's aging population problem."

**速度：0.049秒 / 約780トークン / 15,805 tok/s**

構造化された回答が返ってきた。原因・現状・対策と整理されていて、文法崩壊もない。量子化崩壊は起きていない。

ただし**事実誤認がある。** "workforce is expected to decline by 40% in the next decade"、"pension system is expected to collapse in the next decade"——極端でソースのない主張。コンテンツも表面的で繰り返しが多い。

**英語なら使えるが、品質はお察し。** 8Bモデルの3bit量子化だから当然ではある。

### 速度は本物

| テスト | 生成時間 | トークン/秒 |
|--------|---------|------------|
| 日本語（短い挨拶） | < 0.001s | 19,087 tok/s |
| 日本語（長文） | 0.029s | 15,821 tok/s |
| 英語（長文） | 0.049s | 15,805 tok/s |

公式ブログの17,000 tok/sという数値とおおよそ一致。TTFTもほぼゼロ。

**ゴミが速く出てくるか、まともな回答が速く出てくるかは、言語次第。**

以前ローカルLLMを試したときも、日本語対応の難しさは感じていた。

https://zenn.dev/because02and/articles/lfm25-local-llm-trial

やっぱり日本語は鬼門なのか。なぜこうなるのかClaudeに聞いてみた。

## 何が起きているか

日本語が崩壊する原因は**3bit量子化**にある。

LLMの重み（パラメータ）は通常16bitや32bitの浮動小数点で表現される。これを3bitまで圧縮すると、表現できる精度が大幅に下がる。

**英語が生き残って日本語が死ぬ理由：**

- 英語はトレーニングデータ内でのトークン出現頻度が圧倒的に高い
- そのためモデルの重みに冗長性があり、多少精度が落ちても意味が保持される
- 日本語はトークンの出現頻度が低く、各パラメータがより多くの情報を担っている
- 3bitまで圧縮すると、冗長性のない日本語の内部表現が壊滅する

なるほどわからん。技術的には全く別の話だが、Claude Codeで日本語がRust panicしてた問題を思い出す。

https://zenn.dev/because02and/articles/opencode-oh-my-opencode-setup

「byte index not char boundary」——あれはRustの文字列処理のバグで、量子化とは関係ない。でも「英語圏で作られたツールやモデルにとって、日本語は常に後から対応する存在」という感覚は共通している。

Taalasの公式ブログでは "some quality degradations" とさらっと書いてあるが、日本語に関しては**完全に使えないレベル**だ。

## Taalasが刺さりそうな領域

Taalasはリアルタイムチャットボットや同時通訳など、色々な用途が考えられる。

で、何に使えるかなと考えた。

携帯？スマートウォッチ？家電とか？IoTみたいな感じで、身の回りのデバイス全部にLLMが入ったら面白そうだ。エアコンの音声操作、冷蔵庫の食品認識——やることが固定化されている用途なら、最新モデルじゃなくてもいい。シリコンに焼いてしまえば速度も出るし、低コスト・低電力で相性が良さそうに見える。

でも待て。固定化されているということは、安い汎用チップで十分なのでは。家電に17,000 tok/sはいらない。エアコンの音声操作に爆速推論は完全にオーバーキルだし、安い汎用チップで事足りる。わざわざ専用チップを焼く意味がない。

Taalasの本領はやっぱり**クラウド推論API**にある。

### 本当に刺さるのは「大量ユーザー × 同一モデル」

考え直すと、Taalasが一番素直に活きるのは**大量のユーザーが同じモデルを叩くクラウド推論API**——まさに今Taalasがやっていることだ。

同一モデルに大量リクエストが来る環境では、「1モデル1チップ」の制約がそのまま強みになる。モデルを入れ替える必要がない。速度と電力効率で圧倒的なアドバンテージが出る。

今はエージェントが並列で動く時代になってきている。LLM呼び出しが何十回も連鎖するワークフローでは、1回あたりのレイテンシが積み重なる。速度はますます重要になる。

ただし矛盾もある。エージェントにはある程度賢いモデルが必要だ。3bit量子化の8Bモデルでは、複雑な推論や計画は厳しい。**速いけど賢くない**——ここで面白いのが分業の可能性だ。フロンティアモデル（ClaudeやGPT）が計画を立て、Taalasの高速チップが実行する。賢さと速さの両立を、1つのモデルではなくアーキテクチャで解決するアプローチ。

### モデル陳腐化という最大の弱点

ただし、LLMは数ヶ月で世代交代する。チップを焼くのに約2ヶ月かかる。

Taalas自身も最初の製品であるLlama 3.1 8Bが「すでに最先端ではない」と認めている。この陳腐化リスクをどう管理するかが、Taalasの最大の課題だろう。

## まとめ

**速度は本物。** 15,000〜19,000 tok/sは嘘じゃない。

**品質は言語依存。** 英語なら動く。日本語は完全に壊れる。3bit量子化の限界。

**現状のTaalas = 英語専用の超高速推論エンジン。** ニッチだが、刺さる領域は確実にある。

多言語対応は次世代（4bit以上の量子化）を待つ必要がある。「1モデル1チップ」というアプローチが速度の物理的限界をどこまで押し広げるのか、引き続き注目していきたい。
